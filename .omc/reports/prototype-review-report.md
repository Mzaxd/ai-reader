# AI Reader 原型审查报告

**审查日期**: 2026-02-19
**原型版本**: v1.2 (MotherDuck 风格)
**审查范围**: 阅读器、人物关系、事件分析、时间线、AI问答、书库
**审查角度**: 原型设计层面（非实现细节）

---

## 审查说明

本报告关注**原型设计层面**的问题，不包括：
- Mock 数据硬编码（原型设计的正常状态）
- 按钮空操作（原型只展示 UI）
- 实现细节（localStorage、状态管理等）

---

## 1. 功能设计完整性

### 阅读器模块

| 问题类型 | 描述 | 严重程度 |
|----------|------|----------|
| 功能缺失 | 无**全文搜索**入口 - 用户无法查找内容 | Major |
| 功能缺失 | 无**笔记/高亮**功能设计 - 阅读器标配 | Major |
| 功能缺失 | 无**阅读设置面板**设计 - 字体/字号/主题切换 | Minor |
| 设计缺失 | 联动面板的"人物/时间/事件"Tab 内容未设计 | Minor |

### 人物关系模块

| 问题类型 | 描述 | 严重程度 |
|----------|------|----------|
| 功能缺失 | 无**人物章节索引**设计 - 点击人物应显示所有出场章节 | **Blocking** |
| 功能缺失 | 无**节点详情面板**设计 - 点击节点后显示什么？ | Major |
| 关系类型不全 | 缺少"组织/派系/阵营"关系类型 | Minor |
| 功能缺失 | 无**关系筛选**设计 - 如何按类型/强度筛选？ | Minor |

### 事件分析模块

| 问题类型 | 描述 | 严重程度 |
|----------|------|----------|
| 功能缺失 | 因果链与事件矩阵**无关联设计** - 两个视图如何联动？ | Major |
| 功能缺失 | 无**事件筛选**设计 - 如何按类型/置信度筛选？ | Minor |

### 时间线模块

| 问题类型 | 描述 | 严重程度 |
|----------|------|----------|
| 功能缺失 | 与事件分析页**无联动设计** - 时间线如何跳转到因果链？ | Major |
| 增强功能 | 无**多线索并行视图**设计 - 悬疑多视角、家族多代际 | Minor |

### AI 问答模块

| 问题类型 | 描述 | 严重程度 |
|----------|------|----------|
| 功能缺失 | 无**会话管理**设计 - 会话列表、命名、删除 | Minor |

## 2. 交互流程设计

### 跨模块导航

| 问题 | 描述 | 建议 |
|------|------|------|
| 阅读器 → 人物关系 | 实体标记点击后跳转到哪里？显示什么？ | 应跳转到该人物详情，高亮在关系图中的位置 |
| 阅读器 → 事件分析 | 事件标记点击后跳转到哪里？ | 应跳转到该事件在因果链中的位置 |
| 人物关系 → 阅读器 | "来源章节"如何跳转？ | 应跳转到具体段落并高亮 |
| 时间线 → 阅读器 | "跳转原文"跳到哪里？ | 应跳转到具体段落 |

### 模块内交互

| 问题 | 描述 | 建议 |
|------|------|------|
| 人物关系图 | 节点点击后发生什么？ | 应显示人物详情卡片 + 高亮关联节点 |
| 人物关系图 | 关系线点击后发生什么？ | 应显示关系详情 + 来源章节 |
| 事件矩阵 | 事件行点击后发生什么？ | 应跳转到因果链中对应位置 |
| 因果链 | 原因/中继/结果点击后发生什么？ | 应显示事件详情 + 跳转到原文 |

## 3. 信息架构问题

### 页面结构

| 问题 | 描述 | 建议 |
|------|------|------|
| 阅读器三栏布局 | 在窄屏下中心阅读区过窄 | 应设计折叠/响应式方案 |
| 人物关系面板 | 修正表单占用了大量空间 | 可考虑收起/展开设计 |

### 数据关联设计

| 问题 | 描述 | 影响 |
|------|------|------|
| 实体 ID 命名 | 阅读器用 `actor_ye`，人物页无对应 ID | 需要设计统一的实体 ID 规范 |
| 时间节点 ID | 时间线无 entity ID | 需要设计时间实体与段落的关联 |

## 4. 功能设计遗漏

### 核心功能缺失

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **人物章节索引** | 点击人物应显示所有出场章节，支持一键跳转 | P0 |
| **全文搜索** | 支持关键词搜索，高亮结果，跳转到位置 | P1 |
| **笔记/高亮系统** | 选中文字添加笔记，高亮标注，导出 | P1 |

### 交互设计缺失

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **节点详情面板** | 人物/事件节点点击后显示详情 | P0 |
| **跨模块跳转参数** | 导航时应携带上下文（哪个实体、哪个章节） | P1 |
| **阅读设置面板** | 字体、字号、主题切换 | P2 |

---

## 5. 改进建议（按优先级）

### P0 - 必须补充的设计

1. **人物章节索引** - 核心功能，点击人物显示出场章节列表
2. **节点详情面板** - 人物/事件节点点击后的交互设计
3. **跨模块导航参数设计** - 页面跳转时如何传递实体上下文

### P1 - 重要设计

1. **全文搜索入口** - 阅读器/全局搜索功能设计
2. **笔记/高亮系统** - 阅读标注功能设计
3. **因果链与事件矩阵联动** - 两个视图如何关联

### P2 - 增强设计

1. **多线索并行时间线** - 可选增强功能
2. **会话管理** - AI 问答会话列表
3. **阅读设置面板** - 个性化设置
4. **响应式布局** - 窄屏适配

## 6. 原型设计优点

| 方面 | 优点 |
|------|------|
| 视觉风格 | 统一的 MotherDuck 风格，暖色调阅读体验 |
| 页面结构 | 核心模块划分清晰：阅读器、人物、事件、时间线 |
| 联动面板 | 阅读器右侧联动面板设计合理 |
| 时间归一化 | AI 纠偏建议的交互设计清晰 |
| 因果链视图 | 原因→中继→结果的视觉呈现直观 |

---

## 7. 总结

### 原型设计完整度评估

| 模块 | 完整度 | 主要缺失 |
|------|--------|----------|
| 阅读器 | 70% | 搜索、笔记、设置面板 |
| 人物关系 | 60% | 章节索引、节点详情、筛选 |
| 事件分析 | 75% | 与矩阵联动、筛选 |
| 时间线 | 80% | 与事件页联动 |
| AI 问答 | 85% | 会话管理 |

### 下一步建议

原型阶段应优先补充以下设计：
1. **人物章节索引**的交互设计
2. **节点详情面板**的 UI 设计
3. **跨模块导航**的参数传递设计

这些是核心功能设计，应在原型阶段明确，避免进入开发后返工。
